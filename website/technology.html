Technology

Storage decisions you can audit.

ArgosFS combines a FUSE frontend, Reed-Solomon shards, tier-aware placement, persistent metadata, and a conservative planner.

FUSE POSIX metadata Planner Metadata journal Shard encoder Disk pool

Layered for experimentation, not mystery.

The data plane keeps file bytes in compressed, optionally encrypted stripes. The control plane keeps health, capacity, workload, and policy state inspectable in metadata and JSONL logs.

Subsystems

The moving parts.

Erasure coding

Configurable `k+m` Reed-Solomon layouts distribute data and parity shards across online disks.

Placement policy

Weighted rendezvous scoring considers tier, latency, capacity pressure, NUMA locality, and failure domains.

Metadata integrity

Hash-chained journal records and multiple metadata copies recover cleanly from partial writes.

Advanced I/O

Buffered I/O, direct I/O fallback, io_uring fallback, and mmap-backed read staging are selectable.

L2 cache

Persistent cache writes are indexed and pruned incrementally; L2 hits promote into RAM without rewrites.

Health model

SMART, latency, capacity, and fsck observations drive risk scores and bounded action planning.

Inspectability

Ask the planner before it mutates anything.

argosfs autopilot ROOT --dry-run --explain --json
argosfs health ROOT --json
argosfs verify-journal ROOT
